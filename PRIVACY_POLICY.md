### Overview

This Privacy Policy explains how we handle data when using Google Gemini AI services for production readiness analysis.

### What Data We Collect

#### 1. Repository Information
- **Public Repository URLs**: We only analyze public GitHub repositories
- **Repository Metadata**: Owner, repository name, branch information
- **Commit Information**: Commit hashes for analysis tracking

#### 2. Code Analysis Data
- **File Contents**: Source code files for analysis purposes only
- **Dependencies**: Package.json, requirements.txt, and similar dependency files
- **Configuration Files**: Dockerfiles, CI/CD configurations, README files

### How We Use Your Data

#### 1. AI Analysis Process
- **Code Quality Assessment**: Analyzing code structure and patterns
- **Security Analysis**: Identifying potential security vulnerabilities
- **Performance Evaluation**: Assessing code performance characteristics
- **Best Practices Review**: Checking adherence to industry standards

#### 2. Data Processing
- **Temporary Storage**: Code data is processed in memory only
- **No Persistent Storage**: Source code is not stored after analysis
- **Analysis Results**: Only analysis results are saved to your account

### Data Protection Measures

#### 1. Security Protocols
- **HTTPS Encryption**: All data transmission is encrypted
- **API Key Security**: Google Gemini API keys are securely managed
- **Access Controls**: Limited access to analysis systems

#### 2. Privacy Safeguards
- **No Code Retention**: Source code is not stored after analysis
- **Anonymous Processing**: Analysis is performed without user identification
- **Secure APIs**: All external API calls use secure protocols
